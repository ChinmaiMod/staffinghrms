# Client & Contact Management - UI Design Specification

## ğŸ“‹ Executive Summary

The Client Management module in HRMS provides an interface to manage end clients and client contacts. While the data is stored in the CRM database (`clients` and `contacts` tables), HRMS users can create, view, and update client information directly from the HRMS interface without switching to CRM.

**Data Storage:** CRM Database (single source of truth)
- Clients â†’ `clients` table in CRM
- Client Contacts â†’ `contacts` table in CRM with `contact_type = 'Client Empanelment'`

**Page URLs:**
- List: `/hrms/clients`
- Detail: `/hrms/clients/:clientId`
- Create: `/hrms/clients/new`
- Edit: `/hrms/clients/:clientId/edit`
- Add Contact: `/hrms/clients/:clientId/contacts/new`

**Access Level:** HRMS Admin, HR Manager, Project Manager  
**Primary Function:** Manage end clients and their contacts for employee project assignments

### 8.2 Database Operations

**Creating a Client:**
```sql
-- Insert into CRM clients table
INSERT INTO clients (
  tenant_id, business_id, client_name, client_code, client_type,
  industry, website, address_line_1, address_line_2, city, state, zip_code, country,
  primary_contact_name, primary_contact_email, primary_contact_phone,
  payment_terms_days, notes, is_active
) VALUES (...);

-- If primary contact provided, also insert into CRM contacts
INSERT INTO contacts (
  tenant_id, business_id, client_id,
  first_name, last_name, email, phone, job_title,
  contact_type, -- 'Client Empanelment'
  is_active
) VALUES (...);
```

**Creating a Contact:**
```sql
-- Insert into CRM contacts table
INSERT INTO contacts (
  tenant_id, business_id, client_id,
  first_name, last_name, email, phone, job_title, linkedin,
  contact_type, -- Always 'Client Empanelment' for client contacts
  notes, is_active
) VALUES (...);

-- Update client's primary contact if flagged
UPDATE clients
SET primary_contact_name = :name,
    primary_contact_email = :email,
    primary_contact_phone = :phone
WHERE client_id = :client_id;
```
