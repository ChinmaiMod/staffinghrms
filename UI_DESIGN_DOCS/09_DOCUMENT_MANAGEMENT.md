# Document Management - UI Design Specification

## ğŸ“‹ Executive Summary

The Document Management module handles all document uploads, AI-powered parsing, version control, and document viewing across the universal checklist system. This specification covers the document upload interface, document viewer, and AI parsing results display.

**Page URLs:**
- Document Upload: `/hrms/documents/upload`
- Document Viewer: `/hrms/documents/:documentId`
- Document List: `/hrms/documents` (filtered by entity)

**Access Level:** All employees (own documents), HRMS Admin (all documents)  
**Primary Function:** Upload, parse, track, and view documents with AI assistance

**Security & Access Control:**
- Documents must be access-controlled by entity ownership and RBAC (employees can only access their own documents; admins may access broader scope).
- Authorization must be enforced server-side (Edge Functions/RLS/Storage policies), not only by UI controls.
- Viewing/downloading documents that contain sensitive PII must require appropriate permission; â€œrecent authenticationâ€ may be required for high-risk actions.

## âœ… 6. Validation Rules

### Upload Validations

| Rule | Validation | Error Message |
|------|------------|---------------|
| File Size | Max 10MB | "File size exceeds 10MB limit" |
| File Type | PDF, DOC, DOCX, JPG, PNG, JPEG | "Unsupported file type" |
| File Name | No special characters | "File name contains invalid characters" |
| Duplicate | Warn if same file name exists | "A file with this name already exists. Replace?" |

### Document Metadata Validations

| Field | Validation | Error Message |
|-------|------------|---------------|
| Document Name | Required, 3-255 chars | "Document name is required" |
| Start Date | Optional, valid date | "Invalid date format" |
| Expiry Date | Optional, must be after start date | "Expiry date must be after start date" |

